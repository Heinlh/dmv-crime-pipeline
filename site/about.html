About the Pipeline

Ingestion, storage, and transformation behind this dashboard.

Architecture

Two Python extractors pull incident-level data from each agency's public API: the Socrata SODA API for Montgomery County, and an ArcGIS FeatureServer for Washington DC. Each extractor is incremental, tracking a per-source watermark and re-pulling a 24 hour overlap window to catch late agency corrections. Raw API responses land untouched, as parquet, partitioned by source and extraction date, so the warehouse can always be rebuilt from the landing zone.

DuckDB rebuilds the raw tables from every parquet file landed so far, then a SQL transform dedupes on the source incident id (keeping the latest version), standardizes timestamps and locations, and maps each jurisdiction's offense labels into one shared taxonomy: violent, property, vehicle, drug, society, other, each with a 1-10 severity weight. The result lands in marts.fct_incidents, one row per incident, upserted idempotently on a global incident key. A lightweight export step then snapshots the warehouse into the JSON/GeoJSON this site reads.

Design decisions

  • Raw means raw. The landing zone stores exactly what the APIs return, as strings. Typing and cleaning happen only in SQL, so a source-side type quirk never breaks ingestion.
  • Idempotent everywhere. Raw tables are full rebuilds from parquet; the fact table uses INSERT OR REPLACE on a globally unique incident key. Re-running any step is always safe.
  • Category mapping is transparent. DC's closed set of nine offenses is mapped as explicit data (dim_offense_map); Montgomery County's dozens of NIBRS categories are mapped by documented keyword rules.
  • Locations stay block-level as published. Failed agency geocodes are stored as NULL, never guessed or sharpened.

Data notes

  • Montgomery County: dataMontgomery Crime dataset (icn6-v9z3), recorded crimes since July 2016, UCR/NIBRS classified, preliminary reports subject to change.
  • Washington DC: MPD Crime Incidents feature layers, one per calendar year. Coordinates are anonymized to the block.
  • Both agencies publish with a lag of one or more days. This site says "as of last published data," never "live."
