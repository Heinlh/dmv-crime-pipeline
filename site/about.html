About DMV Crime Watch

A daily-updated view of reported criminal activity across the DMV area.

What is this?

DMV Crime Watch pulls official, public crime reports for the DMV area, currently Washington DC, Montgomery County, MD, Prince George's County, MD, and Fairfax County, VA, into one place, refreshed automatically every day. The Map shows where recent incidents happened, Trends shows how activity has changed over any period back to 2016, Events lets you search individual reports from the last 90 days, and the Daily Brief summarizes the latest data day (also available by email). Arlington County stopped publishing machine-readable incident data in 2022, so it can't be included until the county resumes.

Where the data comes from

  • Montgomery County: the dataMontgomery Crime dataset (icn6-v9z3), recorded crimes since July 2016, UCR/NIBRS classified. Reports are preliminary and subject to change.
  • Washington DC: MPD Crime Incidents feature layers on Open Data DC, one layer per calendar year. Coordinates are anonymized to the block by MPD.
  • Prince George's County: the OpenPGC Crime Incidents datasets (2017 to present). The county updates this data weekly, so PG County incidents trail the daily sources by up to a week. Addresses are rounded to the block by the county.
  • Fairfax County: FCPD's public "Crimes Against" feature services (the data behind the county's own crime mapping dashboard), NIBRS classified and updated hourly by the county. FCPD publishes offense, date, police district, and an offset map point, but no street address, so Fairfax incidents show their district instead of a block address.
  • Every agency publishes with a lag, so this site says "as of last published data," never "live."

How it works

One Python extractor per jurisdiction pulls incident-level records from each agency's public API every morning. Each extractor is incremental: it tracks a per-source watermark and re-pulls a 24 hour overlap window to catch late agency corrections. Raw API responses land untouched as parquet, so the warehouse can always be rebuilt from the landing zone.

A DuckDB warehouse then rebuilds its raw tables, dedupes on each source's incident id (keeping the latest version), standardizes timestamps and locations, and maps each jurisdiction's offense labels into one shared taxonomy: homicide/fatal violence, violent crime, sexual offenses, property crime, vehicle-related crime, drug/alcohol/disorder, and other/unknown. The result is one relational fact table, marts.fct_incidents, one row per incident, upserted idempotently on a global incident key. A final export step snapshots the warehouse into the static JSON this site reads, and GitHub Actions deploys the whole thing daily.

Honest caveats

  • Preliminary data. Agencies revise reports after publishing; the pipeline picks up corrections on later runs, but any given day's numbers can shift slightly.
  • Block-level locations only. Locations stay as published, never sharpened. Incidents whose geocode failed at the source are counted in the statistics but not shown on the map.
  • The category and severity labels are ours. The shared taxonomy and the 1-10 severity weighting are this project's editorial mapping over each agency's own offense labels, built for cross-jurisdiction comparison, not an official police scale. The original agency label is always shown on each incident.
  • Reported crime is not total crime. These are incidents the agencies recorded and chose to publish; reporting practices differ between jurisdictions.
